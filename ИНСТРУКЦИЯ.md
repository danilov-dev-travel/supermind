# SUPERMIND — инструкция по установке и использованию

## Что это

Скилл (навык) для Claude, превращающий его в оркестратора глубокого
стратегического брейншторма. Внутри — 27 систем мышления, собранные в один
конвейер: анкета → позиция → диагноз → структура → разрыв рамки → атака на
идеи → минимальная проба. Подходит для бизнес-задач, бытовых и личных
дилемм, интеллектуальных тупиков.

---

## Часть 1. Установка

### Способ А — Claude Code, автоустановка по ссылке (рекомендуется)

1. Залей содержимое архива в GitHub-репозиторий (репозиторий должен быть
   публичным, либо у устанавливающего должен быть к нему доступ).
2. В Claude Code выполни две команды:

```
/plugin marketplace add ВАШ_ЛОГИН/ИМЯ_РЕПОЗИТОРИЯ
/plugin install supermind@supermind-skills
```

3. Готово. Проверка: набери `/plugin` — supermind должен быть в списке
   установленных; или просто напиши «прогони через Supermind: тест».

**Обновление** после того, как ты изменил файлы в репозитории:

```
/plugin marketplace update supermind-skills
```

и переустанови плагин. Не забудь поднять `version` в
`.claude-plugin/marketplace.json` и `plugins/supermind/.claude-plugin/plugin.json`.

### Способ Б — Claude.ai (веб и мобильное приложение)

1. Возьми файл `dist/supermind.skill`.
2. Отправь его в любой чат с Claude и нажми **Save skill** на карточке
   файла. Либо загрузи через Настройки → Возможности (Capabilities/Skills).
3. Скилл станет доступен во всех твоих чатах. Актуальные названия пунктов
   меню смотри на https://support.claude.com — интерфейс иногда меняется.

### Способ В — Claude Code, вручную без маркетплейса

```
git clone https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПОЗИТОРИЯ
cd ИМЯ_РЕПОЗИТОРИЯ
./install.sh                # для всех проектов (~/.claude/skills)
./install.sh .claude/skills # только для текущего проекта
```

Перезапусти Claude Code — скиллы из этих папок подхватываются автоматически.

### Если что-то не работает

- Маркетплейс не добавляется → проверь, что файл
  `.claude-plugin/marketplace.json` лежит в корне репозитория и репозиторий
  доступен.
- Скилл не появился после установки плагина → очисти кэш
  (`rm -rf ~/.claude/plugins/cache`), перезапусти Claude Code, переустанови.
- В Claude.ai нет кнопки Save skill → загрузка кастомных скиллов может
  зависеть от тарифа/настроек организации, сверься с support.claude.com.

## Часть 2. Как запускать

**Явно:** «Прогони через Supermind: [задача]», «глубокий брейншторм»,
«нужно нестандартное решение», «мы упёрлись», «всё перепробовали».

**Неявно:** просто принеси сложную задачу — бизнесовую, бытовую или личную.
Скилл сам распознает, что нужен полный разбор, а не быстрый совет.

**Когда он НЕ включится (и правильно):** фактические вопросы, задачи одного
шага, просьбы написать текст/код, чистая эмоциональная поддержка.

## Часть 3. Что произойдёт — по шагам

1. **Гейт.** Скилл молча оценит твоё сообщение по 8 блокам сырья (ситуация,
   симптом, факты, что пробовали, ставка, ограничения, «священные коровы»,
   горизонт). Всё есть — едет сразу. Есть дыры — задаст вопросы волнами по
   3–4, с пояснением «зачем», потом соберёт из ответов структурированный
   запрос и покажет тебе на подтверждение.
2. **Разбор.** Определит, с каким типом ситуации имеет дело, найдёт главный
   узел — два сталкивающихся желания/нужды — и место, где малое усилие даёт
   большой сдвиг.
3. **Генерация.** Шесть разных техник бьют по найденному узлу. Норма — от
   12 идей, включая «неприличные».
4. **Атака.** Лучшие идеи проверяются на прочность: «прошёл год, провалилось
   — почему?», игра за противника, «как такое обычно кончается у других».
   Для необратимых решений предложит «холодного критика» — готовый текст,
   который ты вставишь в НОВЫЙ чат и принесёшь вердикт обратно.
5. **Итог.** Понятным языком, без терминов: о чём на самом деле вопрос →
   что происходит → главный узел → 3 проверенных варианта с шансами и
   слабыми местами → что отброшено и почему → первый шаг-проба с сигналами
   «работает/не работает».

## Часть 4. Как получить максимум

**Сильный запрос** сразу закрывает анкету. Шаблон:

```
Прогони через Supermind, полный режим.
СИТУАЦИЯ: [что за дело, масштаб в цифрах]
ФАКТЫ: [3–7 голых наблюдений и цифр, без выводов]
ЧТО ПРОБОВАЛИ: [попытка → результат → почему не сработало]
ПОЗИЦИЯ И СТАВКА: [кто ты, что на кону, что будет через год без действий]
ОГРАНИЧЕНИЯ: [деньги/время/люди; что точно не трогаем]
СВЯЩЕННЫЕ КОРОВЫ — разрешаю трогать: [что можно ставить под сомнение]
ГОРИЗОНТ: [когда нужны первые сигналы]
ВОПРОС: [как ты его видишь — скилл имеет право переформулировать]
```

**Режимы:** «блиц» (≈5 минут, один проход), «стандарт» (≈20–30 минут, со
сверкой карты), «глубокая сессия» (каждая фаза отдельно). Скажи в начале —
или скилл спросит сам.

**Профи-режим:** по умолчанию вывод — простым языком с примерами из жизни.
Хочешь техническую карту (уровни рычага, формулы противоречий, названия
техник) — скажи «покажи по фазам» или «профи-режим».

**Журнал решений.** В конце сессии скилл предложит завести
«журнал-решений.md». Соглашайся: туда пишутся прогнозы и результаты проб,
и со временем скилл калибруется под тебя.

**Возврат пробы.** Сделал пробу — вернись и расскажи, что произошло
(фактами: цифры, эпизоды, дословные фразы). Скилл прогонит результат через
проверку («это случилось бы, даже если идея пустая?»), обновит картину и
запишет в журнал. Три мутных возврата подряд — повод пересмотреть саму
постановку, и скилл это сделает.

**Эмоциональные и личные темы.** Приноси как есть. В остром состоянии скилл
сначала поговорит по-человечески и попросит не принимать необратимых решений
сегодня; разбор предложит, когда схлынет. В личных темах не будет жаргона.

## Часть 5. Что внутри (для любопытных)

- `SKILL.md` — ядро: конвейер, фазы, защиты.
- `references/dossiers/` — 27 досье систем мышления (Cynefin, Медоуз, ТРИЗ,
  Голдратт, Уордли, Тетлок, Щедровицкий и др.), каждое: суть, механика,
  где сильна, где ломается.
- `references/synthesis.md` — почему конвейер собран именно в таком порядке.
- `references/intake.md` — анкета-гейт: волны вопросов и дожимы.
- `references/human-mode.md` — эмоциональный и бытовой режим.
- `references/cold-critic.md` — независимая атака на идеи.
- `references/journal.md` — журнал решений и возврат проб.
- `references/plain-language.md` — правила простого языка.
- `evals/evals.json` — 9 тестов: прогоняй после любых правок скилла.

Работает в паре со скиллами `platforma` (быстрая диагностика — вызывается
как фаза 1) и `llm-council` (совет критиков в фазе атаки), если они
установлены; без них — полностью самостоятелен.
