# Cynefin (Дэйв Сноуден, 1999–наст.)

**Суть.** Ситуации различаются не содержанием, а *режимом причинности*. Очевидные (связь причина→следствие видна), Сложные (скрыта, вскрывается анализом), Комплексные (видна только ретроспективно), Хаотичные (отсутствует), плюс Замешательство (не знаешь, где ты). Каждому режиму — своя стратегия познания и действия.

**Когнитивная операция.** Сопоставить эпистемическую стратегию с текстурой причинности. Не «что делать», а «как здесь вообще узнаётся, что делать».

**Механика.** 1) Декомпозировать задачу на части. 2) Для каждой спросить: как связаны причина и следствие? 3) Присвоить режим. 4) Действовать по режиму: Очевидное — sense→categorize→respond (чек-лист); Сложное — sense→analyze→respond (эксперт); Комплексное — probe→sense→respond (безопасные зонды); Хаос — act→sense→respond (сначала стабилизировать). 5) Следить за границей Очевидное/Хаос — самоуспокоенность обрушивает туда.

**Вход → выход.** Сырая ситуация → карта частей с режимами и предписанным способом действия для каждой.

**Сильна.** Как роутер: предотвращает главную ошибку — экспертный анализ там, где нужны эксперименты, и эксперименты там, где есть готовый ответ.

**Ломается.** Когда используется как ярлык («всё комплексное») без декомпозиции; не даёт содержательного решения — только режим его поиска.

**Уникальный вклад.** Единственная рамка, которая говорит, *какими другими рамками здесь можно пользоваться*. Мета-инструмент, естественный первый шаг любого конвейера.
