# Soft Systems Methodology (Питер Чекленд, 1970-е+)

**Суть.** В «мягких» проблемах спорно само определение проблемы: разные участники живут в разных картинах мира (Weltanschauung). Прежде чем решать — выстроить и сопоставить эти картины.

**Когнитивная операция.** Легализация множественности: проблема — не объект, а переговорная конструкция между позициями.

**Механика.** 1) Rich picture: нарисовать ситуацию со всеми акторами, конфликтами, тревогами — без структуры, как есть. 2) Для каждой значимой позиции — CATWOE: Customers (кто выигрывает/страдает), Actors, Transformation (что во что превращается), Weltanschauung (почему это осмысленно), Owner (кто может остановить), Environment (внешние ограничения). 3) Root definition: «система, делающая X посредством Y ради Z» — для каждой картины своя. 4) Сопоставить модели с реальностью → повестка изменений, которые *желательны и культурно осуществимы*.

**Вход → выход.** Запутанный человеческий клубок → набор эксплицитных картин мира + осуществимая повестка.

**Сильна.** Оргконфликты, реформы, «внедрили систему — саботируют»: там, где провал заложен в несогласованном определении задачи.

**Ломается.** Технические задачи с ясным заказчиком; требует времени и терпения к неоднозначности.

**Уникальный вклад.** W в CATWOE: обязательный вопрос «при какой картине мира это действие разумно?» — противоядие от «они идиоты».
