# Pre-mortem (Гэри Кляйн, 2007; на базе эффекта prospective hindsight)

**Суть.** До старта команда телепортируется на год вперёд в мир, где проект УЖЕ провалился, и пишет историю провала. Исследования (Mitchell, Russo, Pennington): взгляд из «свершившегося» будущего повышает точность генерации причин на ~30% против «что может пойти не так?». Побочный эффект важнее основного: скептикам официально разрешено говорить.

**Когнитивная операция.** Проспективный ретроспективный взгляд: заменить «возможно ли?» на «уже случилось — почему?».

**Механика.** 1) Объяви: «Прошёл год. Проект — катастрофа. Это факт». 2) Каждый молча пишет 3–5 причин провала (молча — чтобы не заражаться и не оглядываться на статус). 3) Собери, кластеризуй; отдельно выдели причины, которые никто не произносил вслух на этапе планирования. 4) Для топ-причин: индикатор раннего обнаружения + превентивный ход. 5) Не превращай в правку плана до неузнаваемости — это стресс-тест, не редизайн.

**Вход → выход.** План → ранжированные сценарии провала + ранние индикаторы + точечные усиления.

**Сильна.** Перед необратимыми шагами; в командах с сильным лидером-энтузиастом (даёт голос сомнению).

**Ломается.** Ритуализация («проведём премортем для галочки»); на стадии генерации идей преждевременна — убивает хрупкие идеи в зародыше.

**Уникальный вклад.** Грамматический трюк с реальным эффектом: прошедшее время («провалился») открывает доступ к причинам, которые будущее время («может провалиться») не достаёт.
